# ANSWER KEY — Day A Assessment: Quality & Non-Conformance

**FACILITATOR ONLY — Do not distribute to participants**

---

| Q | Correct Answer | Option | Explanation |
|---|---------------|--------|-------------|
| 1 | B | QMS-NC-2026-001 | The Nexara format is `QMS-{TYPE}-{YEAR}-{NNN}`. All other options use incorrect delimiters or prefixes. |
| 2 | C | Product | The defect is in the physical product (machined components with a dimensional fault). Category reflects source, not root cause. Process would be correct if the procedure was not followed; Supplier would require an external party. |
| 3 | D | Critical | Critical severity = actual impact on product safety or regulatory compliance. Major = potential impact. The question specifies "actual impact." |
| 4 | B | Documents must be available, protected, and managed for changes | ISO 9001:2015 cl. 7.5.2 and 7.5.3 specify availability, protection, distribution, access, retrieval, use, and preservation. Printing, certification system requirements, and sending to the CB are not required by the standard. |
| 5 | B | Process | The procedure exists but was not followed — this is a process execution failure. System NC would apply if no procedure existed. Product NC would apply if the end result was a defective item independent of the procedure issue. |
| 6 | C | The stage in the value chain where the NC was first identified | Detection point = where in the process the NC was found (incoming, in-process, final, customer, post-delivery). Not root cause, not approver, not customer report date. |
| 7 | C | The immediate steps taken to control and isolate the non-conformance | Containment = emergency response to stop the non-conformance from spreading or causing further harm. Corrective action addresses root cause and prevents recurrence — these are separate. |
| 8 | B | A system constraint — linked CAPA must have status Effective or Closed | This is a hard system constraint in Nexara. Options A, C, and D are not how the system works. |
| 9 | C | Inadequate procedure | The calibration procedure does not exist — this is an inadequate (or missing) procedure root cause. Equipment failure would apply if the instrument itself malfunctioned. Human error would apply if the procedure existed but was not followed. |
| 10 | B | From within the NC record using Actions → Create CAPA | Creating standalone CAPAs breaks the NC-to-CAPA audit trail required for ISO 9001 clause 10.2 evidence. Option A is explicitly incorrect for ISO compliance. |
| 11 | C | A date after all corrective actions are expected to be implemented, allowing time to verify effectiveness | The effectiveness review date must be after implementation, not concurrent or before. 30 days from creation would often be before actions are complete. |
| 12 | C | Raise a new CAPA with a revised root cause analysis | An Ineffective CAPA indicates the root cause was incorrect or the action was insufficient. Do not mark as Effective or close the NC without resolving the underlying cause. |
| 13 | B | NC record + investigation + root cause + corrective action + effectiveness evidence | ISO 9001:2015 cl. 10.2.2 requires: nature of NCs + actions taken + results of corrective action. This maps to the full chain in option B. Minor NCs still require corrective action. |
| 14 | B | 18% of NCs raised in the period are categorised as repeats | Repeat NC Rate = NCs flagged as repeats / Total NCs × 100. 18% is well above the healthy threshold of <10% and indicates systemic corrective action failures. |
| 15 | C | CAPA Effectiveness Rate | CAPA Effectiveness Rate directly measures whether corrective actions are working (i.e., the NC did not recur). Repeat NC Rate is a lagging indicator of the same thing but measured differently. |
| 16 | B | Apply filters: Category = Supplier, Status = Open, Due Date = Past on the NC module list view | Built-in filters in the list view are the correct tool. Exporting to Excel is inefficient and bypasses the portal's real-time data. |
| 17 | C | A significant scope change requiring full re-approval | In semantic versioning, major version increment = breaking change or significant scope change requiring full re-approval. Minor (.x) = small update; same approver route. |
| 18 | B | 8.4 — Control of externally provided processes, products and services | Supplier failures map to clause 8.4 (external provision control). Clause 8.7 covers internal nonconforming outputs. Clause 10.2 covers the corrective action response, not the categorisation of source. |
| 19 | B | A clause-mapped list of records grouped by ISO 9001:2015 requirement with links to each record | The ISO evidence package index is a navigable, clause-mapped document. It does not contain user lists, audit findings, or standard text. |
| 20 | B | Document version history report + approval workflow log + superseded document archive | ISO 9001 cl. 7.5 evidence requires proof of: document creation/update (version history), controlled distribution (approval log), and management of obsolete documents (superseded archive). |
