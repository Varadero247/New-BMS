# Nexara IMS Administrator Training Programme — Overview

**Programme Code**: NEXARA-ATP-001
**Version**: 1.0
**CPD Hours**: 14 hours contact time (2 full days)
**Format**: Instructor-Led Training (ILT) — available in-person or virtual
**Maximum Cohort Size**: 16 participants
**Minimum Cohort Size**: 4 participants

---

## Executive Summary

The Nexara IMS Administrator Training Programme is a two-day, role-based training course designed to equip client-side platform administrators with the knowledge and practical skills required to configure, maintain, govern, and troubleshoot the Nexara Integrated Management System (IMS).

On completion, participants receive a **Nexara Certified Platform Administrator** certificate, confirming they are qualified to independently administer the IMS within their organisation.

### Prerequisites

Participants should arrive with:
- Working knowledge of their organisation's IMS modules (user-level experience)
- Basic understanding of user account management in any enterprise system
- Familiarity with web-based administration consoles
- No programming experience required

---

## Programme Aims

By the end of this two-day programme, participants will be able to independently administer the Nexara IMS platform across all seven core administration domains, applying Nexara's documented best practices and security standards.

---

## Learning Outcomes (by Module)

### Module 1 — User Management & SCIM Provisioning
Participants will configure the full user lifecycle including SCIM 2.0 auto-provisioning, bulk import, and identity provider integration.

### Module 2 — Role & Permission Configuration
Participants will design and implement a role matrix across 39 predefined roles, 17 modules, and 7 permission levels using the RBAC framework.

### Module 3 — Module Activation & Configuration
Participants will activate, configure, and manage dependencies between IMS modules using the module registry.

### Module 4 — Integration Management
Participants will create and manage API keys, configure OAuth 2.0 clients, implement SAML SSO, and set up webhooks.

### Module 5 — Audit Log Review
Participants will navigate the audit log architecture, interpret the event taxonomy, filter for compliance-relevant events, and export audit data.

### Module 6 — Backup & Restore Procedures
Participants will execute pg_dump backups, automate backup schedules, perform restore procedures, and run the disaster recovery runbook.

### Module 7 — Platform Update Management
Participants will manage the update lifecycle through staging gates, execute rollbacks, and control feature flag activation.
