# Virtual Delivery Guide — Train-the-Trainer and Nexara Programmes

**Programme**: Nexara IMS Train-the-Trainer (T3) and all downstream programmes
**Version**: 1.0
**Platforms**: Zoom and Microsoft Teams (primary); other platforms on request

---

## 1. Overview

Virtual delivery requires more deliberate facilitation than in-person delivery. Without physical presence, you must work harder to maintain engagement, manage group dynamics, and create the psychological safety that adults need to participate actively. This guide covers the platform setup, facilitation techniques, assessment invigilation, and common problems for virtual Nexara training sessions.

## 2. Platform Requirements

### Minimum Technical Requirements (Trainer)

| Requirement | Specification |
|-------------|-------------|
| Camera | 1080p preferred; 720p minimum |
| Microphone | Headset with noise cancellation strongly recommended; built-in laptop mic is acceptable for quiet rooms only |
| Internet | Stable broadband ≥ 10 Mbps upload |
| Second screen | Strongly recommended — facilitator on Screen 1, participant gallery on Screen 2 |
| Backup device | Smartphone or tablet as backup if primary device has issues |
| Platform | Zoom (version 5.13+) or Microsoft Teams (current); meeting host licence required |

### Minimum Technical Requirements (Participants)

| Requirement | Specification |
|-------------|-------------|
| Camera | Must be enabled for the full session; participants without functioning cameras cannot join assessed delivery |
| Microphone | Must be functioning; mute when not speaking |
| Internet | ≥ 5 Mbps; if on Wi-Fi, position close to router |
| Browser | For Nexara Training Portal: Chrome (latest) or Edge (latest) |
| Dual tab access | One tab for the virtual meeting, one for the Nexara sandbox/Training Portal |

### Pre-Session IT Check

Schedule a **30-minute IT test call** 2 business days before the session. All participants must attend:

| Check | Confirmed? |
|-------|-----------|
| Video is working | ☐ |
| Audio is clear (trainer can hear participant; participant can hear trainer) | ☐ |
| Screen share is working (trainer shares screen; participants confirm visibility) | ☐ |
| Nexara sandbox loads in participant browser | ☐ |
| Training Portal assessment loads (if running assessment) | ☐ |
| Breakout rooms function (if using; test with at least 2 participants) | ☐ |

Participants who cannot pass the IT check should attempt to resolve before Day 1. If unresolved, contact your IT department; Nexara cannot provide IT support for participant devices.

## 3. Meeting Setup

### Zoom Setup

| Setting | Configuration |
|---------|-------------|
| Meeting ID | Use a fixed meeting ID for both days; resend to participants if they lose the link |
| Waiting room | Enable — admit participants individually; allows you to be set up before the first arrival |
| Co-host | Assign a co-host (a participant acting as technical assistant) — they manage the waiting room, breakouts, and chat |
| Recording | Obtain explicit consent before recording; Nexara does not require recordings; do not record assessments |
| Chat | Enable; monitor chat throughout; designate a chat moderator if running a large session |
| Screen share | Trainer only by default; enable participant screen share for sandbox demonstration exercises |
| Annotation | Disable — participants annotating on your screen during delivery is distracting |
| Reactions | Enable — use for quick polls (thumbs up/down) |

### Microsoft Teams Setup

| Setting | Configuration |
|---------|-------------|
| Meeting type | Scheduled meeting (not Teams channel meeting) — provides better control |
| Lobby | Enable lobby for participants; admit individually |
| Recording | Same as Zoom — consent required; do not record assessments |
| Background | Set your background; encourage participants to use virtual background or blur if their environment is distracting |
| Breakout rooms | Set up breakout rooms before the session; Teams requires room creation at meeting start |
| Chat | Monitor throughout |
| Participant microphone/video | You can mute/unmute participants as needed from the host view |

## 4. Virtual Facilitation Techniques

### Opening

Start 5 minutes before the scheduled time. Greet participants as they join. Use the waiting room to manage entry. When all participants are present:

> "Good morning everyone — can you all see my screen? And can you hear me clearly? Great. Let's get started."

Ask everyone to turn their camera on at the opening: "For today to work well, I need to be able to see you — just as you'd expect in a face-to-face session. Cameras on, please, throughout the day."

### Maintaining Engagement

Virtual participants disengage more quickly and less visibly than in-person participants. Compensate:

| Technique | Application |
|-----------|------------|
| **Nominal group** | Before any question, use a poll or chat prompt: "Type your answer in the chat first — everyone types at the same time before sending." This prevents the first answer from anchoring all others. |
| **Directed naming** | As in in-person delivery, address specific participants by name. Do not rely on volunteers in virtual settings — the default is silence. |
| **Reaction polls** | Use Zoom/Teams reactions for quick temperature checks: "Thumbs up if you've used this feature before." Visible without requiring unmuting. |
| **Breakout rooms** | Use breakout rooms of 2–3 for pair and trio discussions. 5–8 minute breakouts work well. Brief before sending: "Discuss X; one person will share back." |
| **Chat channel** | Ask participants to post questions and thoughts in chat throughout. Monitor chat continuously; address periodically (every 20 min at minimum). |
| **Camera check** | If a camera goes off for more than 5 minutes, privately message the participant: "Just checking — are you still with us?" |

### Managing Virtual Energy

Energy management is harder virtually but follows the same principles as in-person:

| Signal | Virtual Response |
|--------|----------------|
| Declining chat activity | Launch a chat prompt: "30-second chat challenge: what is the single most useful thing you've learned so far? Go." |
| Cameras off | Announce: "I'm going to ask everyone to camera on for the next exercise — this one needs everyone visible." |
| No volunteers for questions | Switch from open to directed: "[Name], I'd love your view on this." |
| Extended passive listening | Switch to a breakout: "Let's break into pairs for 5 minutes to discuss this." |

**The 90-minute rule**: Never run a virtual presentation session for more than 90 minutes without a substantive interactive break (not just a 2-minute camera-off break — a genuine group task or breakout).

## 5. Breakout Room Management

### Setup

- Create breakout rooms before the session or at session start
- Assign participants manually (don't use random assignment — you want specific pairings)
- Prepare the breakout task before sending participants: display the task on screen and in chat before opening rooms

### Typical Breakout Formats

| Format | Group Size | Duration | Instruction |
|--------|-----------|---------|------------|
| Pair discussion | 2 | 5 min | "Discuss [topic]. One of you will share back." |
| Trio case study | 3 | 8 min | "Read the scenario in your handout and identify the correct NC category. Agree on an answer." |
| Group exercise | 3–4 | 10 min | "Build a 3-month training schedule for the scenario described." |

### Monitoring Breakouts

- Use "Visit" function (Zoom) or join breakout rooms directly (Teams) to check on progress
- Do not interrupt immediately — give groups 2 minutes to get started
- Broadcast a "2 minutes remaining" message to all rooms before closing

### Reconvening

Give 30 seconds' notice before closing breakout rooms. When all participants return:
> "Welcome back. Let's hear from each group — [Name from Group 1], what did your group find?"

## 6. Virtual Assessment Administration

### Configuring the Portal Assessment

The process is identical to in-person (see `02-day-2-delivery-certification/MODULE-ASSESSMENT-DELIVERY.md`), except:

- Launch the assessment from the Training Portal admin view (separate browser tab)
- Share the assessment link in the meeting chat after giving the pre-assessment briefing
- Confirm all participants have clicked the link and can see the first question before starting the timer

### Virtual Invigilation Protocol

| Requirement | Virtual Specification |
|-------------|----------------------|
| Cameras | All participants cameras ON for the full assessment |
| Screen share (optional) | Participants share their screen (Training Portal assessment only) to the main meeting |
| Chat | Disable participant-to-participant chat during assessment |
| Microphones | All muted; participants raise hand in platform if they have a technical issue |
| Trainer visibility | Stay on camera; monitor gallery view throughout |
| Timer | Share a timer on screen or announce the time remaining at 15 min and 5 min marks |

### Common Virtual Assessment Problems

| Problem | Solution |
|---------|---------|
| Participant's portal freezes | Ask them to refresh the browser; if assessment resets, contact Nexara immediately |
| Participant's camera goes off | Private message: "Please keep your camera on during the assessment." |
| Participant uses external device (phone visible) | Quietly message: "Please put your phone away during the assessment." Note for records. |
| Meeting drops for one participant | They rejoin; the timer does not stop; note the interruption time |
| Meeting drops entirely | Reconnect; confirm remaining time; continue from where you stopped |

## 7. Post-Session

After each virtual session:

- Confirm all participants have received their assessment result via email or private message
- Issue certificates via Training Portal (same process as in-person)
- Export the chat log if useful for the Parking Lot Q&A you promised to address
- Submit completion data to Nexara within 5 business days
- Collect feedback via portal survey (prompt participants in the session closing)
