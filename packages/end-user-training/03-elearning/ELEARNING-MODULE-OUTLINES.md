## Module 2: Recording Incidents (~22 slides, 40 min)

### Learning Objective
Record an incident or near miss with all required information.

### Slide Sequence

| Slide | Type | Content |
|-------|------|---------|
| 02-01 | Title | Module 2: Recording Incidents |
| 02-02 | Learning objectives | 3 objectives |
| 02-03 | Scenario intro | "It's Monday morning. As you arrive, you witness a near miss in the car park..." |
| 02-04 | Drag-and-drop | Categorise 4 event descriptions: Incident / Near Miss / Observation — immediate feedback |
| 02-05 | Narrated content | Why reporting matters: "Every near miss is a warning sign. Reporting prevents the next injury." |
| 02-06 | Click-reveal | Incident form — 8 mandatory fields annotated with hotspots |
| 02-07 | Narrated content | Field: `title` — good vs bad examples shown ("Slip on wet floor near Entrance B" vs "Slip") |
| 02-08 | Narrated content | Field: `dateOccurred` — must be when it happened, not when reported |
| 02-09 | Decision branch | "Which severity should you select for a hospitalised colleague?" → MAJOR → explanation |
| 02-10 | Narrated content | Fields: location, immediateActions, description |
| 02-11 | Try-it | Simulated form: complete 5 fields for the car park near miss scenario |
| 02-12 | Narrated content | Adding evidence: photos, witness statements |
| 02-13 | Try-it | Upload a simulated photo to the evidence tab |
| 02-14 | Narrated content | What happens after submission: status tracking, investigation process |
| 02-15 | Scenario outcome | "Your report triggered an investigation. Here's what happened next..." |
| 02-16 | Common mistakes | 4 click-reveal cards: wrong date, wrong type, no photo, delay in reporting |
| 02-17 | Summary | Key takeaways |
| 02-18 | Knowledge check (5 Q) | Module 2 knowledge check set |
| 02-19 | Results | Score + feedback; proceed to Module 3 |
